Var:

Variables declared with 'var' are called function-scoped. It means if they are declared in a function then they are only visible inside the function. We can't access the variables outside the function. If 'var' is declared outside the function then it becomes a global variable which can be accessible inside and also outside of any function.


'var' declarations are hoisted to the top of their scope. which means they are moved to the top during the compilation phase and we can use it before declaration but the value will be undefined until it's declaration.



Let:

Variables declared with 'let' are called block-scoped. It means they are only visible within the block {} where they are defined. 'let' also becomes global variable when it is declared outside of any curly braces {}.


'let' declarations are not hoisted to top of their scope. I means we can't use it before declaration. Unless it will throw an error.



Const:

'const' is also a block-scoped like 'let'. But we use 'const' to declare a constant variable. It means its value will never be modified. But if we declare any array or object with 'const' then we can modify their elements but cant't reasign the variable.

Null:

We assign 'null' to a variable to intentionally indicate that the variable has no value. 'null' can be assigned to an object value to intensionally represent the absence of the value. It can be used as placeholder for an object that has not been initialized. We often use it to reset an object variable.



Undefined:

If we declared a variable but didn't assign any value to it then, the value of the variable is 'undefined'. The parameters of a function that were not passed are 'undefined'. If a function returns nothing then the result of the function will be 'undefined'.

REST API:

REST stands for Representational State Transfer, and API stands for Application Programming Interfaces. REST is an architectural style for designing networked applications. RESTful APIs are the set of constraints and principles applied to the web services that adhere to the REST architecture.


Web service exchange data between client and server through APIs. RESTful web services follow the REST architecture. Each request from the client to the server must contain all the needed information to understand the request. Resources are the main abstractions in REST. Resources are uniquely identified by an URI (Uniform Resource Identifier). We can manipulate the resources using standard HTTP methods like GET, POST, PUT, and DELETE. RESTful APIs have an uniform and consistatent interface. This inclueds a consistent way to address resources through URIs. A consistant way to represent resources using standard media types such as JSON, or XML.
